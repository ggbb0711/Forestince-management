# Implementation Plan: Facility Top Bookers & Remove Reports Page

**Branch**: `002-facility-top-bookers` | **Date**: 2026-03-18 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/002-facility-top-bookers/spec.md`

## Summary

Add a "Top Bookers" leaderboard (top 10 users by booking count) to the facility Usage tab, and remove the Reports page and all associated server/client code. The top bookers data is fetched as part of the existing facility stats endpoint — no new route needed. The reports feature is surgically removed: 3 server routes + service functions deleted, entire `reports/` client directory deleted, sidebar link + router entry removed.

## Design Decisions

### Top Bookers — Server

The `getFacilityStats()` function is extended to run one additional query inside its `Promise.all`:

```typescript
prisma.booking.groupBy({
  by: ['userId'],
  where: { facilityId, startTime: { gte: from, lte: to } },
  _count: { id: true },
  orderBy: { _count: { id: 'desc' } },
  take: 10,
})
```

After the groupBy, resolve user names/companies with `prisma.user.findMany({ where: { id: { in: userIds } }, include: { company: true } })`. Assign ranks 1–N, break ties alphabetically.

`FacilityStatBreakdown` gains a `topBookers: TopBooker[]` field. `TopBooker` is defined as:
```typescript
interface TopBooker {
  rank: number
  userId: string
  userName: string
  companyName: string
  bookingCount: number
}
```

### Top Bookers — Client (FacilityUsageTab)

A new "Top Bookers" card is rendered below the utilization rate card. It contains a ranked list: rank badge (number circle), user name + company name, booking count. Skeleton rows shown while loading (same pattern as rest of app). Empty state if `topBookers.length === 0`.

### Reports Removal — Server

`reportService.ts`: delete `getUsageReport`, `getFinancialReport`, `getDamageReport`, `getPeriodLabel`. Keep `getFacilityStats`, `getDefaultDateRange`, `toISODateString`.

`reports.ts` route: delete `/usage`, `/financial`, `/damage` handlers. Keep `/facility-stats/:id`. Remove dead imports (`ReportGroupBy`, `VALID_GROUP_BY`, 3 deleted service functions).

`messages.ts`: remove `REPORTS` key. `index.ts`: no change — the reports router stays mounted at `/api/reports` for the facility-stats sub-route.

### Reports Removal — Client

Delete `src/features/reports/` entirely. Delete `src/hooks/useFilteredPagination.ts`. Update router, sidebar, breadcrumbs. Move `FacilityStatBreakdown` type into `facility.ts`.
